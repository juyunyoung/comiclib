
## 1. 화면 정의

### 1) 메인(대시보드)

- 나의 만화책 리스트(표지/제목/평점/간단 소감 표시)
- 상단에 각 화면으로 이동하는 네비게이션 바(만화책 등록 화면, 만책 상세 보기화면, 통계화면)
- 간단 통계(읽은 만화 수, 1순위 인물 등)
- 제목, 작가, 등장인물 기준 검색(가운데 입력창 )


### 2) 만화책 등록/수정

- 만화 제목, 작가, 표지 이미지 업로드
- 읽은 날짜, 평점(별점), 완독 여부
- 100자 이내 소감, 체크박스(좋았던 점, 아쉬웠던 점)
- 등장인물 란(추가/삭제), 각 인물에 순위 및 한줄 코멘트 입력


### 3) 만화책 상세 보기

- 등록 정보(제목·작가·이미지·평점·소감)
- 내가 매긴 등장인물 순위(1~5위, 코멘트 포함)
- ‘수정’ 버튼, ‘공유’ 버튼


### 4) 내 통계 화면

- 내가 읽은 만화 기록 통계(막대그래프 또는 리스트)
- 1순위로 선정한 등장인물 TOP5 등


## 2. 필요 기술

### 프론트엔드

- **React.js** (빠른 제작, 컴포넌트 기반 UI)
- **CSS Framework**: Material-UI, TailwindCSS, Chakra UI (간단하고 현대적인 디자인)
- 이미지 업로드, Drag \& Drop, Responsive 지원


### 백엔드

- **Node.js + Express**: REST API 신속 구축
- **간편 데이터베이스**: MongoDB(Atlas) 또는 Firebase Firestore (도입·관리가 쉬움)
- **사용자 인증**: Firebase Auth(구글/이메일 연동 쉬움)
- 만화책 DB는 OMDB/ComicVine API 연동(혹은 수기로 입력)


### 기타 기능/배포

- **Vercel/Netlify**: 프론트 배포(무료/간편)
- **Render/Heroku**: 백엔드(무료 플랜 있음)
- **이미지 업로드**: Cloudinary, Firebase Storage(간편 연동)
- **SNS 공유**: react-share 라이브러리 등
- **간단한 통계**: 프론트에서 데이터 집계(혹은 Chart.js)

**최소 기능 MVP 목표라면:**

1. React 기반으로 한정된 화면(리스트, 등록/상세, 검색)부터 시작
2. 사용자 인증·이미지 업로드·DB 연결은 Firebase 조합 활용
3. 나중에 디자인 고도화, 공유/통계 등 확장

필요한 UI 설계나 기능 모듈 예시, 구현 코드 기본 구조 등 더 구체적으로 원하시면 추가 안내해드릴 수 있습니다!
